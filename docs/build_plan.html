claude_freelance — Build Plan

Multi-platform freelance proposal optimizer • Generated 2026-03-28 • freelance.visionquest2020.net

Overview

What it does: Upload your resume, paste (or auto-receive) job postings from 5 freelance platforms, and get AI-generated tailored proposals via the Claude API — delivered to your inbox for phone review.

Target Platforms

PlatformInput MethodNotes
UpworkEmail alerts + manual pastePrimary platform, bidding model
WellfoundEmail alerts + manual pasteStartup ecosystem, direct hire
ContraManual paste (initially)Portfolio-driven, 0% commission
Turing.comEmail alerts + manual pasteAI-matched, long-term contracts
Freelancer.comEmail alerts + manual pasteBidding + contest model

Two Operating Modes

  1. Manual mode (Phase 1): Log in, paste a job posting, get a proposal generated on screen
  2. Automated mode (Phase 3): Gmail forwards job alerts → server parses → Claude API generates → proposal emailed to you

Database Schema

-- Users (admin auth)
CREATE TABLE users (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    email VARCHAR(255) UNIQUE NOT NULL,
    password VARCHAR(255) NOT NULL,
    role ENUM('admin','user') DEFAULT 'admin',
    is_active TINYINT(1) DEFAULT 1,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

-- Resumes
CREATE TABLE resumes (
    id INT AUTO_INCREMENT PRIMARY KEY,
    user_id INT NOT NULL,
    title VARCHAR(255) NOT NULL,
    content TEXT NOT NULL,                -- plain text version for API
    file_path VARCHAR(500),              -- uploaded file (PDF/DOCX)
    is_active TINYINT(1) DEFAULT 1,      -- which resume to use for generation
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id)
);

-- Platforms
CREATE TABLE platforms (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100) NOT NULL,          -- Upwork, Wellfound, etc.
    slug VARCHAR(50) UNIQUE NOT NULL,
    base_url VARCHAR(255),
    is_active TINYINT(1) DEFAULT 1,
    alert_email_from VARCHAR(255),       -- sender address for email parsing
    parser_class VARCHAR(100),           -- e.g., UpworkParser
    notes TEXT
);

-- Jobs (parsed job postings)
CREATE TABLE jobs (
    id INT AUTO_INCREMENT PRIMARY KEY,
    user_id INT NOT NULL,
    platform_id INT NOT NULL,
    external_id VARCHAR(255),            -- platform's job ID if available
    title VARCHAR(500) NOT NULL,
    description TEXT NOT NULL,
    skills_required TEXT,                -- comma-separated or JSON
    budget_min DECIMAL(10,2),
    budget_max DECIMAL(10,2),
    budget_type ENUM('fixed','hourly','not_specified') DEFAULT 'not_specified',
    client_info TEXT,                    -- client name, rating, history
    job_url VARCHAR(500),
    source ENUM('manual','email') DEFAULT 'manual',
    raw_email TEXT,                      -- original email body if from pipeline
    fit_score TINYINT,                   -- 1-10 AI-assessed match quality
    fit_notes TEXT,                      -- why it's a good/bad fit
    status ENUM('new','proposal_drafted','submitted','rejected','won','archived') DEFAULT 'new',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id),
    FOREIGN KEY (platform_id) REFERENCES platforms(id)
);

-- Proposals (generated drafts)
CREATE TABLE proposals (
    id INT AUTO_INCREMENT PRIMARY KEY,
    job_id INT NOT NULL,
    resume_id INT NOT NULL,
    content TEXT NOT NULL,               -- the generated proposal text
    tone VARCHAR(50),                    -- corporate, casual, technical
    suggested_rate DECIMAL(10,2),
    rate_type ENUM('hourly','fixed','not_specified'),
    version INT DEFAULT 1,              -- regeneration tracking
    is_submitted TINYINT(1) DEFAULT 0,
    submitted_at TIMESTAMP NULL,
    api_model VARCHAR(100),             -- which Claude model was used
    api_tokens_used INT,
    generation_time_ms INT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (job_id) REFERENCES jobs(id),
    FOREIGN KEY (resume_id) REFERENCES resumes(id)
);

-- Proposal Rules (do's and don'ts for the AI)
CREATE TABLE proposal_rules (
    id INT AUTO_INCREMENT PRIMARY KEY,
    user_id INT NOT NULL,
    category ENUM('always','never','tone','skills','availability','rate','custom') DEFAULT 'custom',
    rule_text TEXT NOT NULL,
    is_active TINYINT(1) DEFAULT 1,
    sort_order INT DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id)
);

-- Availability (calendar scheduling)
CREATE TABLE availability (
    id INT AUTO_INCREMENT PRIMARY KEY,
    user_id INT NOT NULL,
    available_from DATE NOT NULL,
    available_to DATE,                   -- NULL = indefinitely
    hours_per_week INT DEFAULT 40,
    notes TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id)
);

-- Error Log
CREATE TABLE error_log (
    id INT AUTO_INCREMENT PRIMARY KEY,
    message TEXT NOT NULL,
    context TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

Build Phases

Phase 1 — Foundation + Manual Mode Core

Get the app running with manual paste-and-generate workflow.

  1. Scaffold MVC framework — core/, config/, public/index.php, .htaccess files, .env
  2. Database setup — create freelance DB, run all migrations
  3. Auth system — login, logout, register, forgot/reset password, session guards
  4. Admin dashboard — left sidebar nav, stats overview (jobs received, proposals generated, win rate)
  5. Resume management — upload PDF/DOCX, extract plain text, CRUD, set active resume
  6. Platform management — seed 5 platforms, enable/disable, configure parser class
  7. Proposal rules — CRUD for rules by category, drag-and-drop reorder, seed defaults from spec:
    • Honest about skills (MySQL primary, not Postgres)
    • 26 years experience
    • Availability from April 1, 2026
    • Rate suggestion based on budget range
    • Flag weak fits — don't draft for bad matches
    • Match tone to client (corporate vs casual)
  8. Manual job entry — paste job posting, select platform, parse into structured fields
  9. Claude API integration — ClaudeApiService.php + ProposalGenerator.php:
    • Build prompt: resume + job + rules + availability → system/user message
    • Call Claude API, get proposal text + fit score + suggested rate
    • Store result in proposals table
  10. Proposal view/edit — display generated proposal, allow edits, copy-to-clipboard, regenerate with different tone, print/PDF export
  11. Apache vhost — freelance.local pointing to public/
  12. Git init + .gitignore + first commit

Phase 2 — Platform Parsers + Job History Enhancement

Build the per-platform parsers and job tracking pipeline.

  1. Job parsers — one class per platform in app/Services/Parsers/:
    • UpworkParser.php — extract title, description, skills, budget, client info from Upwork job format
    • WellfoundParser.php — startup-style postings, equity info
    • FreelancerParser.php — bidding format, contest detection
    • TuringParser.php — long-term role format, tech stack matching
    • ContraParser.php — portfolio-style project descriptions
  2. Job history & search — paginated list, filter by platform/status/fit score, full-text search
  3. Proposal history — all generated proposals, version tracking, status (drafted/submitted/won/lost)
  4. Availability calendar — visual calendar for scheduling availability windows, hours per week
  5. Dashboard analytics — proposals per platform, win rate, average fit score, API usage/cost

Phase 3 — Automated Email Pipeline Automation

The hands-free mode: job alerts arrive, proposals appear in your inbox.

  1. Inbound email setup — configure jobs@visionquest2020.net (or similar) on the server
  2. Gmail forwarding — filters for each platform's alert emails, forward to server address
  3. Email ingestion — IMAP poll (simpler, scheduled via cron) or pipe-to-script:
    • Connect to mailbox, fetch UNSEEN messages
    • Identify platform from sender/subject
    • Route to correct parser
  4. Auto-generate pipeline — EmailPipeline.php:
    • Parse email → create Job record
    • Check fit score — skip if below threshold
    • Generate proposal via Claude API
    • Email draft via claude_messenger
    • Log everything for review in dashboard
  5. Cron job — poll every 5-15 minutes (configurable)
  6. Email reply integration — register in claude_mailbox poller for phone-based approve/edit/reject

Phase 4 — Polish & Production Deploy

  1. PDF generation — export proposals as formatted PDFs (DOMPDF)
  2. Image processing — resume file thumbnails, profile images
  3. Security hardening — CSRF, CSP headers, input sanitization, rate limiting on API calls
  4. PHPUnit tests — models, services, parsers, API integration
  5. Deploy script — scripts/deploy.sh for git-pull deployment to AWS
  6. Production .env — wire up RDS credentials, Anthropic API key, mail config
  7. SSL setup — Let's Encrypt for freelance.visionquest2020.net
  8. Nightly automation — .claude-nightly config, PHPUnit + Playwright overnight runs
  9. Cost estimate — baseline run via claude_estimator

Core Engine: How Proposal Generation Works

The Prompt Architecture

The ProposalGenerator assembles a Claude API call with:

ComponentSourcePurpose
System promptHardcoded + rules DBDefine the AI's role as a proposal writer, inject all active proposal rules
ResumeActive resume from DBSkills, experience, background for the AI to reference
Job postingParsed job recordWhat the client wants — title, description, skills, budget
AvailabilityCalendar DBCurrent availability window and hours/week
Platform contextPlatform recordAdjust tone/format for Upwork bid vs Wellfound intro vs Contra pitch

Output Structure

The AI returns a JSON object with:

  • fit_score (1-10) — how well the job matches your skills
  • fit_notes — why it's a good/bad fit
  • proposal_text — the full proposal draft
  • suggested_rate — recommended rate based on budget + your experience
  • tone — detected/applied tone (corporate, casual, technical)
  • skip_reason — if fit_score is too low, why it recommends not bidding
